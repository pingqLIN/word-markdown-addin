# Skill List

這份清單整理目前對 `word-markdown-addin` 最有實際價值的 skills。

它不是完整全域 skill registry，而是這個 repo 維護時最常用的子集。

## GitHub / 撰文相關

- `doc-coauthoring`
  - 適合整理 `README`、安裝說明、release notes、維護文件。

- `internal-comms`
  - 適合撰寫變更摘要、交接說明、公告式更新、對內說明稿。

- `gh-address-comments`
  - 適合處理 GitHub PR review comments、issue 回覆、需要逐條回應的討論串。

這三個是目前最直接對應「GitHub 撰文 / repo 文件整理」的技能。

## UI / 前端

- `frontend-design`
  - 用於 taskpane 版面重配、資訊層級重組、按鈕與互動區塊整理。

- `webapp-testing`
  - 用於 taskpane 頁面與其他 localhost 頁面的前端互動驗證。

## 測試 / 品質

- `project-development-loop`
  - 適合做發布前整理、進度盤點、風險檢查、收尾 review。

## 環境 / 能力盤點

- `env`
  - 用於確認目前 session 真的有哪些 skills、apps、plugins、MCP servers 可用。

## 對本專案目前最相關的 skill mapping

- `README / docs / release notes 更新`
  - 優先使用 `doc-coauthoring`

- `GitHub 上逐條回應 comment`
  - 優先使用 `gh-address-comments`

- `對內交接 / 狀態更新 / 發佈摘要`
  - 優先使用 `internal-comms`

- `taskpane UI redesign`
  - 優先使用 `frontend-design`

- `taskpane smoke test`
  - 優先使用 `webapp-testing`

- `發布前整理檢查`
  - 優先使用 `project-development-loop`

- `確認目前 skill / plugin / app 是否存在`
  - 優先使用 `env`

## 使用提醒

- 某些 skills 可能已安裝在本機，但不一定每個 session 都會暴露在可直接呼叫的清單裡。
- 如果需要確認某個 skill 這一輪能不能真的用，先用 `env` 或先檢查目前 session 的 skills inventory。
