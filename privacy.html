Word Markdown Companion

這份說明只涵蓋目前公開線上版的資料處理行為。這個專案沒有自建帳號系統，也不要求你把 Markdown 內容上傳到專案自管的後端；核心操作是在 Word task pane 與使用者裝置上完成。

快速安裝 支援頁 GitHub Repo

這個 add-in 會處理哪些資料

你主動匯入的 Markdown 檔案

當你使用 Import .md 或拖放匯入時，檔案內容會在 task pane 內讀取，並插入目前的 Word 文件。

目前的 Word 文件內容

當你使用 Export .md 或格式轉換功能時，task pane 會讀取目前文件內容，將它轉成 Markdown 供預覽、複製或下載。

本機 UI 偏好

公開版 task pane 會用 localStorage 保存 theme 與 font scale 設定，以便下次開啟時沿用。

剪貼簿與下載

只有在你主動點擊複製或另存功能時，才會寫入系統剪貼簿或觸發下載。

公開線上版與單機 helper 的差異

  • 公開線上版不需要 Windows registry 覆寫，也不依賴 localhost bridge。
  • 單機 helper 模式可能會使用本機 .local/pending-open.json 與 .local/taskpane.log，那是 Windows 本機開發/測試流程，不是公開站點的預設路徑。
  • 公開 GitHub Pages 站點本身不新增專案自管的分析追蹤端點。

補充說明

  • GitHub Pages 或其底層平台仍可能依其自身政策處理標準網站請求紀錄。
  • 如果你不想保留本機 UI 偏好，可清除瀏覽器儲存資料或移除 add-in。
  • 如果你對公開版資料處理有疑問，請改用 https://github.colorgeek.co/word-markdown-addin/support.html 提供的支援入口。
